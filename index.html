Work Philosophy

  • Know what you don't know

    Every challenge is surmountable. We just need to identify what is not known yet but needs to be known, then seek advice from the experts.

  • Safety first, not even production can come before safety

    Management shall make it clear that staff must not be punished when production is halted due to safety concerns.

  • Single source of truth

    Every data must be stored only once at one place. All other resources that uses the same data must refer to the original source. For example, a high level trip setpoint is stored in a safe operating limit (SOL) database. If an operating procedure requires mentioning the setpoint, it must link to the database, rather than rewriting it. Should the setpoint be revised, the procedure will be automatically updated as well.

  • No data destruction

    All data must never be deleted; if digital housekeeping is necessary, it must be archived only. All data contains insights. Even errors such as typo gives insights on human error and how to detect/fix them.

  • Seek general solutions; if a similar problem reoccurs, use previous solutions and improve.

    For example, if a project undergoes an economics feasibility study, do a general template with consistent assumptions. If a new project comes, use the template and base it against the same principles as previous projects.
